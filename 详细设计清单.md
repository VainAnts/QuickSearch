2. 核心技术设计
2.1 清单文件 (manifest.json) 关键配置
json
{
  "manifest_version": 3,
  "name": "QuickSearch",
  "version": "1.0.0",
  "permissions": ["contextMenus", "storage", "activeTab"],
  "host_permissions": ["https://www.baidu.com/*", "https://www.bing.com/*", "https://www.google.com/*"],
  "background": {
    "service_worker": "background.js"
  },
  "action": {
    "default_popup": "popup.html",
    "default_icon": {
      "16": "icons/icon16.png",
      "48": "icons/icon48.png",
      "128": "icons/icon128.png"
    }
  },
  "icons": { ... },
  "commands": {
    "_execute_action": {
      "suggested_key": {
        "default": "Ctrl+Shift+S"
      },
      "description": "Search selected text"
    }
  }
}
2.2 右键菜单与搜索逻辑 (background.js)
初始化菜单：在扩展安装/启动时，动态创建右键菜单项（例如“Search with Default”）。

获取选中文本：监听 runtime.onInstalled 和 contextMenus.onClicked，通过 chrome.tabs.sendMessage 或 chrome.scripting.executeScript 获取当前选中的文本。

执行搜索：根据用户存储的默认引擎，组装搜索URL并调用 chrome.tabs.create({ url: searchUrl })。

监听存储变化：当用户更新默认引擎时，动态更新菜单项的名称（例如显示“Search with Bing”）。

2.3 配置界面 (popup.html + popup.js)
UI布局：

搜索引擎列表（展示名称、URL模板）。

“添加引擎”按钮（弹出对话框输入名称和URL，其中%s代表关键词）。

每个引擎项旁边有“设为默认”、“编辑”、“删除”按钮。

存储提示（例如“配置已保存并同步”）。

数据存储：

默认数据：[{ name: "Baidu", url: "https://www.baidu.com/s?wd=%s", isDefault: true }, ...]

使用 chrome.storage.sync.set/get 存取。

URL编码：用户输入的关键词需进行 encodeURIComponent 编码后替换URL中的%s。

javascript
const searchEngines = [

3. 关键实现步骤（开发顺序）
创建 manifest.json，声明必要权限。

实现 background.js 中的右键菜单生成和基础搜索（硬编码百度）。

实现 popup.html/js 的引擎管理界面，并存储到 chrome.storage.sync。

修改 background.js，从存储读取默认引擎，动态构建搜索URL。

添加快捷键支持（在 commands 中定义，监听 commands.onCommand 事件）。

增加图标、样式和用户提示（可选通知）。

跨浏览器测试（Edge与Chrome商店的本地加载测试）。

4. 测试要点
功能测试：

选中文本后右键菜单是否正确显示搜索项。

点击菜单项是否在新标签页打开正确搜索结果。

添加/删除/修改搜索引擎后，存储与UI是否同步。

切换默认引擎后，右键菜单项的名称是否更新。

兼容性测试：

在最新版Edge和Chrome中分别加载未打包的扩展，验证所有功能。

边界情况：

未选中文本时右键菜单应无搜索项（或置灰）。

URL模板中无%s或编码异常时的处理。

网络断开时点击搜索应仅打开空搜索页（浏览器自身处理）。

5. 本地使用
不需要发布到商店，手动加载扩展。
